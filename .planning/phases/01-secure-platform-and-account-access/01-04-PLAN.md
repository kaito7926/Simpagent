<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Bearer token -> validator | Attacker-controlled JOSE data enters cryptographic/semantic parsing |
| Claims -> PostgreSQL principal | Signed snapshots meet mutable authoritative account state |
| Provider assertion -> account linker | Future provider identities may attach to local accounts |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-12 | Spoofing | HIGH - blocking | JWT validator | mitigate | Fixed RS256/type/key/profile and adversarial corpus |
| T-01-13 | Elevation | HIGH - blocking | principal/policy | mitigate | DB reload, exact state equality, closed enums |
| T-01-14 | Spoofing | HIGH - blocking | account linker | mitigate | Unique issuer/subject, never external email linking |
| T-01-15 | Information Disclosure | HIGH - blocking | auth outcomes/evidence | mitigate | Generic outcomes, bounded event fields, canary scans |
| T-01-SC | Tampering | HIGH - blocking | crypto dependencies | mitigate | Approved PyJWT/pwdlib only; no custom JOSE/KDF |
</threat_model>
