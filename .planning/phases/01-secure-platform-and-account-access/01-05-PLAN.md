<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Browser cookie jar -> session API | Ambient cookies enter state-changing endpoints |
| Raw token -> digest lookup | A bearer secret is transformed without storage |
| Concurrent requests -> PostgreSQL locks | Competing refreshes must serialize and commit evidence |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-16 | Spoofing | HIGH - blocking | refresh storage | mitigate | 256-bit token, keyed digest only, external key |
| T-01-17 | Tampering / Elevation | HIGH - blocking | refresh concurrency | mitigate | Fixed row-lock order and one-child lineage |
| T-01-18 | Spoofing | HIGH - blocking | Origin/CSRF | mitigate | Exact origin plus family-bound signed token |
| T-01-19 | Repudiation | HIGH - blocking | replay denial | mitigate | Revocation and event commit before HTTP mapping |
| T-01-20 | Information Disclosure | HIGH - blocking | cookies/logs/events | mitigate | HttpOnly refresh, bounded evidence, canary scans |
</threat_model>
