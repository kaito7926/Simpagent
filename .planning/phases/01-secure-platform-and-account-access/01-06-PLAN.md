<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Backend outcomes -> browser state | Generic auth/session results control credential memory and UI |
| `/me` -> identity rendering | Server-authoritative role/scopes become visible |
| Cookie jar -> JS | Only CSRF may be readable; refresh must remain inaccessible |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-21 | Information Disclosure | HIGH - blocking | browser storage | mitigate | Memory-only access, no JWT decode/persistence |
| T-01-22 | Elevation | HIGH - blocking | identity rendering | mitigate | `/me` only, known role/scope allowlists |
| T-01-23 | Denial of Service | MEDIUM | refresh client | mitigate | Single-flight and one retry |
| T-01-24 | Repudiation | HIGH - blocking | logout UI | mitigate | Preserve state on failure; claim success only after server success |
| T-01-25 | Spoofing | HIGH - blocking | session errors | mitigate | Generic expiry/replay/inactive/unknown copy |
</threat_model>
