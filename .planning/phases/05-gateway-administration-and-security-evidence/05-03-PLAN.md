<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> FastAPI GitHub OAuth routes | Untrusted browser traffic begins and completes GitHub OAuth flow. |
| FastAPI -> GitHub token/email endpoints | External provider data determines whether linking/provisioning is safe. |
| provider identity -> local account model | Ambiguous provider identity could otherwise attach to the wrong local account. |
| readiness API -> auth shell | Backend capability truth controls GitHub CTA rendering. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Elevation of Privilege | `backend/app/identity/oauth_service.py` | mitigate | Resolve existing OAuth users by stored `(issuer, subject)` first and allow auto-link only for verified normalized email match per D-06 and D-07. |
| T-05-07 | Spoofing | `backend/app/identity/providers/github.py` | mitigate | Require a verified GitHub email before linking or provisioning and deny ambiguous identities per D-08. |
| T-05-08 | Information Disclosure | OAuth error paths and evidence | mitigate | Extend canary-secret tests so provider tokens, secrets, and raw payloads are redacted from responses and evidence snippets per D-21. |
| T-05-09 | Repudiation | account linking decisions | mitigate | Keep deterministic deny branches and corresponding tests so unsafe provider identities create no silent or partial links. |
</threat_model>
