<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> Kong | Untrusted public traffic reaches the gateway first. |
| Kong -> FastAPI | Gateway headers, correlation IDs, and JWT screening affect what the backend receives. |
| upstream proxy -> app | Trusted proxy headers can alter client IP, origin assumptions, and audit evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-14 | Denial of Service | `kong/kong.yml` public routes | mitigate | Add route-specific rate limiting and request-size limits per GATE-03 and GATE-04 before traffic reaches FastAPI. |
| T-05-15 | Spoofing | trusted proxy / forwarded headers | mitigate | Accept forwarded identity data only from configured trusted proxies per D-23; validate correlation IDs and client-IP sources in app middleware. |
| T-05-16 | Elevation of Privilege | Kong JWT screening | mitigate | Keep Kong JWT plugin coarse-only and preserve backend principal resolution and scope/ownership checks per GATE-05. |
| T-05-17 | Information Disclosure | public service exposure | mitigate | Keep Kong Admin API, PostgreSQL, worker control plane, and sandbox control plane off public ports per GATE-06. |
</threat_model>
