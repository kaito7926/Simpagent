## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| database/log evidence -> admin services | Sensitive operational data is transformed before it crosses into admin contracts. |
| Kong config/evidence -> backend service | Backend interprets gateway evidence without treating Kong-only denials as authoritative application persistence. |
| admin contracts -> later route/UI consumers | Only sanitized summaries/snippets are allowed to cross to subsequent layers. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-18 | Information Disclosure | `backend/app/identity/redaction.py` and admin schemas | mitigate | Recursively redact secrets, tokens, cookies, raw prompts, raw grounding payloads, sandbox internals, and host/container identifiers before serialization per D-20. |
| T-05-19 | Repudiation | gateway-only Kong denials | mitigate | Represent Kong-only evidence from config-backed sources and never fabricate FastAPI database rows per D-14. |
| T-05-20 | Tampering | gateway evidence interpretation | mitigate | Keep gateway-evidence parsing bounded, read-only, and driven by explicit Kong-backed sources rather than mutable client data. |
| T-05-21 | Information Disclosure | admin evidence drawers | mitigate | Keep tables summary-only and drawers limited to bounded sanitized snippets with canary-secret coverage per D-18, D-19, and D-21. |
</threat_model>
