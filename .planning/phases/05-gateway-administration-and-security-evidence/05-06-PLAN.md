<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| database/log evidence -> admin API | Sensitive operational data is transformed before it crosses into admin responses. |
| Kong log volume -> backend service | Backend reads gateway evidence from a shared log source without treating it as authoritative application persistence. |
| admin API -> browser drawer views | Only sanitized summaries/snippets cross to the browser. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-18 | Information Disclosure | `backend/app/identity/redaction.py` and admin schemas | mitigate | Recursively redact secrets, tokens, cookies, raw prompts, raw grounding payloads, sandbox internals, and host/container identifiers before serialization per D-20. |
| T-05-19 | Repudiation | gateway-only Kong denials | mitigate | Represent Kong-only evidence from config/log sources and never fabricate FastAPI database rows per D-14. |
| T-05-20 | Tampering | read-only Kong log evidence mount | mitigate | Mount Kong logs read-only into backend and bound parsed line count/path via validated config. |
| T-05-21 | Information Disclosure | admin evidence drawers | mitigate | Keep tables summary-only and drawers limited to bounded sanitized snippets with canary-secret coverage per D-18, D-19, and D-21. |
</threat_model>
