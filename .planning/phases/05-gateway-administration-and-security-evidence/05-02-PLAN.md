<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> FastAPI OAuth routes | Untrusted browser traffic starts and completes the provider redirect flow. |
| FastAPI -> Google OAuth endpoints | Provider token exchange uses environment-held secrets and returns external identity claims. |
| OAuth provider identity -> local account/session model | External identity proof is linked to internal user accounts and session issuance. |
| readiness API -> auth shell | Backend capability truth controls whether the Google CTA is shown or actionable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03 | Spoofing | `backend/app/api/routes/auth_oauth.py` | mitigate | Validate OAuth `state` on callback per D-04, keep redirect/callback fully backend-owned, and deny callback reuse or missing state. |
| T-05-04 | Elevation of Privilege | `backend/app/identity/oauth_service.py` | mitigate | Reuse existing first-party session issuance and persist identity by `(issuer, subject)` per D-05 and D-06 instead of trusting mutable email as the durable key. |
| T-05-05 | Information Disclosure | readiness and auth UI | mitigate | Expose only configured/unconfigured provider capability state; never surface client secrets or provider payloads in readiness, errors, or browser-visible redirects. |
| T-05-SC | Tampering | Python package install path | mitigate | Install Authlib only after Plan 01 approval and pin the reviewed version in `backend/pyproject.toml`. |
</threat_model>
