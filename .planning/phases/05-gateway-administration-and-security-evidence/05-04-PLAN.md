## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> admin API | Untrusted client requests administrative data and write controls. |
| admin shell -> orchestration writes | UI actions can change trusted-supervisor and guardrail state only when authorized. |
| backend metrics -> browser | Aggregate metrics must remain bounded and content-safe when serialized. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Elevation of Privilege | `backend/app/services/admin_evidence.py` | mitigate | Keep `admin:read` and `admin:write` enforcement centralized per D-17 and record denied reads/writes for under-scoped users. |
| T-05-11 | Information Disclosure | `backend/app/schemas/admin.py` metrics payload | mitigate | Restrict Overview to bounded aggregate counters only per OBS-07; do not serialize raw user content or credentials. |
| T-05-12 | Repudiation | orchestration control writes | mitigate | Record orchestration write events with correlation IDs and changed fields so admin actions are auditable. |
| T-05-13 | Tampering | frontend admin shell | mitigate | Treat UI gating as secondary only; backend remains authoritative for every Overview and Orchestration request per AUTHZ-02. |
</threat_model>
