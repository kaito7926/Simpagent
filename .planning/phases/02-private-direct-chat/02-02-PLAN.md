<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| FastAPI settings -> provider adapter | Secret provider credentials and model configuration enter executable adapter code |
| Chat adapter -> OpenAI-compatible provider | Owner-checked conversation text will leave the app once 02-03 calls the adapter |
| Provider -> adapter | Untrusted provider success and failure responses become application state inputs |
| Package registry -> backend runtime | The openai package becomes executable dependency code |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-02-05 | Information Disclosure | HIGH - blocking | provider settings and adapter errors | mitigate | Use SecretStr accessors, never log API keys/tokens/cookies/prompts, and map provider failures to safe ChatProviderError metadata |
| T-02-06 | Tampering | HIGH - blocking | provider request payload | mitigate | Unit-test exact payload allowlist: non-streaming messages only, with no tool/search/file/code-execution parameters |
| T-02-07 | Spoofing | HIGH - blocking | provider success state | mitigate | Treat empty/malformed/status/error provider responses as ChatProviderError and leave persistence decisions to 02-03 |
| T-02-08 | Denial of Service | MEDIUM | provider latency and SDK retries | mitigate | Bounded timeout and low SDK retry ceiling from Settings |
| T-02-SC | Tampering | HIGH - blocking | openai package install | mitigate | Blocking human legitimacy gate before manifest change or install because RESEARCH.md lacks Package Legitimacy Audit table |
</threat_model>
