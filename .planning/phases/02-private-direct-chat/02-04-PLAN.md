<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Browser workspace -> FastAPI | User-entered chat content and generated client_message_id cross into protected API routes |
| Browser session controller -> chat API helpers | Memory-only access token is attached to chat requests |
| API error -> browser UI | Provider and auth failures become user-visible rows or session states |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-02-10 | Information Disclosure | HIGH - blocking | frontend session handling | mitigate | Reuse AuthSessionController.authorizedJson; assert no localStorage token path |
| T-02-11 | Tampering | HIGH | client_message_id generation | mitigate | Generate per-submit UUID and preserve it across retry/error state |
| T-02-12 | Denial of Service | MEDIUM | composer pending behavior | mitigate | Disable send for the same conversation while assistant status is pending |
| T-02-13 | Information Disclosure | MEDIUM | UI error display | mitigate | Show user-safe error copy and correlation ID only; no raw provider bodies |
| T-02-SC | Tampering | MEDIUM | frontend dependencies | accept | No package-manager dependency is added in this plan |
</threat_model>
