<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Browser/API client -> FastAPI chat routes | Untrusted tokens, scopes, cursors, and conversation IDs enter protected routes |
| FastAPI service -> PostgreSQL | Conversation ownership and deletion state become durable |
| Principal claims -> current DB state | Token claims must be checked against current user role, scopes, and active state |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-02-01 | Information Disclosure | HIGH - blocking | conversation read paths | mitigate | Require chat:read, owner-constrain ID and user_id in SQL, hide deleted rows, and test two-user BOLA denial |
| T-02-02 | Tampering | HIGH - blocking | create/delete routes | mitigate | Require chat:write before mutation and update only rows matching id + owner + deleted_at IS NULL |
| T-02-03 | Repudiation | MEDIUM | delete operation | mitigate | Preserve deleted_at soft-delete state and correlation-bearing error envelope |
| T-02-04 | Spoofing | HIGH - blocking | stale principal state | mitigate | Reuse resolve_principal so stale role/scope/account state fails closed before chat logic |
| T-02-SC | Tampering | MEDIUM | package supply chain | accept | No package-manager dependency is added in this plan; later install plans carry blocking legitimacy gates |
</threat_model>
