## Trust Boundaries

| Boundary | Description |
|---|---|
| Sidebar/drawer UI -> chat API | Conversation IDs and cursors from browser state drive protected reads and mutations |
| Delete undo -> database | A hidden conversation can be restored only within the owner-scoped recovery path |
| Account menu -> auth session | Sign-out exits the protected session from the chat workspace |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-02-14 | Information Disclosure | HIGH - blocking | conversation pagination | mitigate | Cursor contains only ordering fields, list is owner-scoped, and tests prove no cross-user rows |
| T-02-15 | Tampering | HIGH - blocking | delete and undo-delete routes | mitigate | Require chat:write and update only id + owner + expected deleted_at state |
| T-02-16 | Spoofing | MEDIUM | mobile drawer/account actions | mitigate | Reuse current authenticated session state and AccountAccessShell logout flow |
| T-02-17 | Denial of Service | LOW | repeated pagination/delete UI actions | mitigate | Disable in-flight buttons and preserve stable layout states |
| T-02-SC | Tampering | MEDIUM | package supply chain | accept | No package-manager dependency is added in this plan |
</threat_model>
