## Trust Boundaries

| Boundary | Description |
|---|---|
| Retrieved search content -> coordinator and UI | Untrusted external content reaches the product |
| Durable metadata -> audit/UI rendering | Persisted search data must remain allowlisted and safe |
| Compose public stack -> private services | End-to-end verification exercises real exposure and degraded paths |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-03-13 | Elevation | HIGH - blocking | prompt injection / tool chaining | mitigate | Treat all retrieved content as untrusted and refuse internal fetches or second tools |
| T-03-14 | Information Disclosure | HIGH - blocking | persistence / logs | mitigate | Retention allowlist and secret-leakage tests |
| T-03-15 | Repudiation | HIGH - blocking | state/evidence | mitigate | Persist every search lifecycle state with correlation IDs |
| T-03-16 | Denial of Service | HIGH - blocking | failure handling | mitigate | Distinct timeout/unavailable/failure states and explicit retry path |
</threat_model>
