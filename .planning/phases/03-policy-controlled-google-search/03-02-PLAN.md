<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Coordinator -> worker | Public API state crosses into a dedicated tool-execution boundary |
| Settings -> provider capability check | Operator-configured model and credentials become runtime decisions |
| Internal credential -> worker validation | The worker must accept only a short-lived audience-bound capability token |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-03-05 | Elevation | HIGH - blocking | worker/tool boundary | mitigate | Dedicated ADK search worker with no Python or arbitrary tools |
| T-03-06 | Spoofing | HIGH - blocking | capability credential | mitigate | Short-lived audience-bound internal worker token, never user bearer reuse |
| T-03-07 | Denial of Service | HIGH - blocking | budgets/timeouts | mitigate | Input/output/time limits and no silent multi-call retry |
| T-03-08 | Tampering | HIGH - blocking | startup capability check | mitigate | Config-driven model plus explicit support/metadata-shape probing |
</threat_model>
