| Boundary | Description |
|---|---|
| Backend search envelope -> client state machine | Search-state distinctions must remain honest in the UI |
| Trusted suggestion/source payload -> rendered UI | Search evidence must not become a raw HTML injection path |
| User interaction -> retry/prefill actions | Suggestion and retry actions must not mutate history incorrectly |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-03-09 | Information Disclosure | HIGH - blocking | evidence rendering | mitigate | Dedicated source/suggestion components, no raw provider HTML in markdown |
| T-03-10 | Tampering | HIGH - blocking | state machine | mitigate | Distinct explicit response-state mapping and retry-in-place behavior |
| T-03-11 | Elevation | HIGH - blocking | suggestion interaction | mitigate | Prefill-only suggestion behavior, never auto-submit |
| T-03-12 | Repudiation | HIGH - blocking | degraded-state UI | mitigate | Honest visible distinction among grounded, denied, unavailable, failure, and timeout states |
