## Trust Boundaries

| Boundary | Description |
|---|---|
| Environment config -> provider resolver | Untrusted configuration values become runtime provider-selection input |
| Coordinator -> Firecrawl client | Search requests cross from policy-owned backend code into an external provider boundary |
| Provider result -> persistence layer | External search metadata is normalized before durable storage and user-visible link rendering |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-03-17 | Elevation | provider resolver | mitigate | Enforce D-10 allowlist in `config.py` and `provider_status.py`; reject any non-allowlisted provider before worker dispatch |
| T-03-18 | Spoofing | Firecrawl-unconfigured path | mitigate | Enforce D-11 in capability and failure tests so missing Firecrawl credentials return `search_unavailable` instead of hidden Gemini fallback |
| T-03-19 | Information Disclosure | Firecrawl persistence | mitigate | Enforce SRCH-05 via `grounding.py`, `chat_turns.py`, and `test_search_retention_allowlist.py` so only reviewed fields persist and no click tracking is stored |
| T-03-20 | Tampering | provider runtime boundary | mitigate | Keep D-14 guardrails: same `tool:websearch` policy gate, capability token, one-tool-per-turn limit, and no arbitrary fetch path |
| T-03-21 | Denial | provider failure matrix | mitigate | Provider-specific readiness and failure-state tests keep invalid provider, unconfigured Firecrawl, timeout, and provider-failed states distinct |
</threat_model>
