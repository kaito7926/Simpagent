## Trust Boundaries

| Boundary | Description |
|---|---|
| Browser turn request -> conversation route | Untrusted mode, prompt, and conversation identifiers enter the search flow |
| Authenticated principal -> policy gate | User claims are re-evaluated immediately before tool execution |
| Service -> database persistence | Search state becomes durable evidence in messages and tool executions |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-03-01 | Elevation | HIGH - blocking | route/policy gate | mitigate | Explicit mode allowlist and immediate `tool:websearch` recheck |
| T-03-02 | Tampering | HIGH - blocking | state persistence | mitigate | One service-owned turn flow and correlated message/tool-state writes |
| T-03-03 | Spoofing | HIGH - blocking | worker seam | mitigate | Injectable worker protocol with no user bearer forwarding |
| T-03-04 | Information Disclosure | HIGH - blocking | grounded/degraded mapping | mitigate | Honest state enum and no fake grounded evidence on missing grounding |
</threat_model>
