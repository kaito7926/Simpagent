<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Backend -> supervisor | Trusted internal execution request boundary |
| Supervisor -> runtime container | Trusted control plane launches untrusted code |
| Runtime -> host/network | Runtime must not gain host, network, package, or command authority |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-04-04 | Elevation | HIGH - blocking | runtime networking | mitigate | `network=none`, side-effect denial tests, no internal-network joins |
| T-04-05 | Elevation | HIGH - blocking | runtime privilege profile | mitigate | non-root, read-only root, dropped caps, no-new-privileges, seccomp |
| T-04-06 | Tampering | HIGH - blocking | package/command surface | mitigate | fixed runner, no pip, no arbitrary subprocess, blocked policy tests |
| T-04-07 | Elevation | HIGH - blocking | supervisor input surface | mitigate | typed request validation and no caller-controlled runtime options |
</threat_model>
